3) Funcionalidades Já Implementadas
3.1) Cadastro e Venda de Passagem
VenderPassagemController:

Tela que permite inserir uma nova passagem (ou editar), salvando campos como:
numBilhete, dataViagem, nomePassageiro, numeroDoc, etc.
Calcula automaticamente alguns valores (alimentação, transporte) de acordo com a rota e o tipo de passagem.
Insere ou atualiza no banco (tabela passagens) via PassagemDAO.
PassagemDAO (CRUD da tabela “passagens”):

inserir(Passagem p), atualizar(Passagem p), listarTodos(), excluir(...).
Já garante o retorno de todos os campos, inclusive valor_pago, valor_total, devedor, etc.
3.2) Lista de Passageiros (ListaPassagensNovo)
Tela para filtrar e visualizar passagens já emitidas.
Pode filtrar por data, viagem, pagamento (em aberto, quitados, etc.), rota, agente, tipoPassageiro.
Exibe as passagens em um TableView.
(Ponto pendente: algumas colunas de pagamento não aparecem do jeito esperado, ver item 4).
3.3) Estrutura de Filtros no DAO
filtrarPassagens(LocalDate dtIni, LocalDate dtFim, boolean somenteViagem, String viagemSel, String pagamento, String rotaSel, String agenteSel, String tipoPassSel)
Retorna as passagens conforme as condições, montando uma query dinâmica.
